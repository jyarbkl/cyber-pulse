# Cyber Pulse

A personal cybersecurity RSS reader hosted on GitHub Pages. Feeds are fetched automatically by GitHub Actions twice daily and served as a static web app — no backend, no proxy, no extensions required.

**Live reader:** `https://[yourusername].github.io/cyber-pulse`

---

## How it works

1. GitHub Actions runs on a schedule (6:00 AM and 11:00 AM CDT)
2. It fetches all RSS feeds listed in `feeds.json` and scrapes the SANS Blog
3. Results are saved to `feeds-data.json` in this repo
4. `index.html` reads `feeds-data.json` and renders the reader in your browser

---

## Features

- **Latest view** — all articles newest first across all sources
- **By Source view** — articles grouped by publication
- **Bookmarks** — save articles to read later (stored in this browser only)
- **13 sources** — curated cybersecurity news, research, and advisories

## Managing feeds

To add or remove a feed, edit `feeds.json` directly in GitHub's web interface:

1. Open `feeds.json` in this repo
2. Click the pencil (edit) icon
3. Add or remove a feed entry
4. Commit the change
5. The next scheduled run will pick it up automatically

Each feed entry follows this format:

```json
{
  "name": "Source Name",
  "url": "https://example.com/feed.xml",
  "type": "rss"
}
```

Use `"type": "scrape"` only for the SANS Blog entry.

---

## Running the workflow manually

To fetch fresh articles on demand without waiting for the schedule:

1. Go to the **Actions** tab in this repo
2. Click **Fetch RSS Feeds** in the left sidebar
3. Click **Run workflow** → **Run workflow**

## File structure

```
cyber-pulse/
├── index.html              # The reader UI
├── feeds.json              # Your list of RSS feed URLs — edit this to add/remove feeds
├── feeds-data.json         # Auto-generated by GitHub Actions — do not edit manually
├── fetch_feeds.py          # Python script run by the workflow
├── README.md               # This file
└── .github/
    └── workflows/
        └── fetch-feeds.yml # GitHub Actions workflow — runs twice daily
```

## Bookmarks

Bookmarks are saved in your browser's `localStorage`. They are:

- Specific to the browser and device you're using
- Lost if you clear your browser data
- Not synced across devices

---

## Known limitations

- Articles are up to ~12 hours old depending on when you visit relative to the last fetch
- SANS Blog is scraped rather than fetched via RSS — if SANS redesigns their site, the scraper may need updating
- The GitHub Pages URL is publicly accessible to anyone who knows it (no sensitive data is exposed)
